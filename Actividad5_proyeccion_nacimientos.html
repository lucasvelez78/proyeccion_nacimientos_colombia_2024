DEFINICIN DE VARIABLES

La prediccin del nmero de nacimientos puede enriquecerse con mltiples factores demogrficos, sociales y temporales. Dado que el proyecto utiliza datos disponibles para los aos 20202023, se seleccionan variables que son consistentes, comparables y disponibles para todos los departamentos. El modelo se basa en una estructura temporal de los datos (ao, mes), en la distribucin por departamentos, y en la relacin histrica del nmero de nacimientos a lo largo del tiempo. Es por esto que la matriz de correlacin no juega un papel muy importante, en nuestro caso, para la definicin de variables.

  • Variable dependiente (salida): Nacimientos

Variable numrica continua (Es la variable que se desea predecir para el ao 2024.)

  • Variables independientes (entrada)

A continuacin, se seleccionan variables relevantes, realistas y adecuadas para el enfoque del proyecto.

  1. Ao

Numrica. Indica el ao al que corresponde el registro (2020, 2021, 2022, 2023). Ascapturamos tendencias temporales (crecimiento, descenso, recuperacin post-pandemia).

  1. Departamento

Categrica. Cada departamento tiene caractersticas demogrficas distintas (densidad poblacional, fecundidad, urbanizacin). Permite al modelo aprender las diferencias estructurales entre territorios.

  1. Nacimientos rezagados (variables lag)

  • Nacimientos del ao anterior (Lag 1): Numrica. Los nacimientos presentan alta autocorrelacin temporal, la cifra de un ao depende fuertemente del ao anterior.

  • Nacimientos de dos aos antes (Lag 2): Numrica. Permite capturar tendencias suaves, ciclos o cambios estructurales. Ayuda a estabilizar el modelo cuando hay fluctuaciones abruptas.

# Definir variables (X = entrada, y = salida)

categorical_features = ["DEPARTAMENTO", "GENERO"]
numeric_features = ["ANO", "MES"]
target = "NACIMIENTOS"

X = df_agg[categorical_features + numeric_features]
y = df_agg[target]


# Separar entrenamiento y validacin

X_train, X_val, y_train, y_val = train_test_split(
    X, y, test_size=0.25, random_state=42
)


# Preprocesamiento (OneHot + Imputacin)

preprocessor = ColumnTransformer(
    transformers=[
        ("num", SimpleImputer(strategy="median"), numeric_features),
        ("cat", Pipeline([
            ("imputer", SimpleImputer(strategy="most_frequent")),
            ("encoder", OneHotEncoder(handle_unknown="ignore"))
        ]), categorical_features),
    ]
)

MODELOS

Para este proyecto se eligieron dos modelos de Machine Learning Regresin Lineal y Random Forest Regressor debido a la naturaleza del problema y al tipo de datos disponibles.

  1. Regresin Lineal

La Regresin Lineal se seleccion como modelo base porque es un modelo adecuado cuando se busca analizar la relacin entre variables como ao, mes, y departamento.

  1. Random Forest Regressor

El modelo Random Forest se eligi porque es capaz de capturar patrones ms complejos que la regresin lineal, incluyendo relaciones no lineales entre variables. Maneja mejor la variabilidad entre departamentos. Random Forest suele obtener un mayor desempeo y se convierte en una excelente opcin para modelar patrones reales en los nacimientos.

# Evaluacin

def evaluar(modelo, nombre):
    pred = modelo.predict(X_val)
    mae = mean_absolute_error(y_val, pred)
    rmse = mean_squared_error(y_val, pred) ** 0.5
    r2 = r2_score(y_val, pred)

    print(f"\n--- {nombre} ---")
    print("MAE:", mae)
    print("RMSE:", rmse)
    print("R:", r2)

    return mae, rmse, r2

eval_lr = evaluar(lr_model, "Regresin Lineal")
eval_rf = evaluar(rf_model, "Random Forest")

# Elegir mejor modelo (menor RMSE)
best_model = rf_model if eval_rf[1] < eval_lr[1] else lr_model
print("\nModelo seleccionado:",
      "Random Forest" if best_model == rf_model else "Regresin Lineal")
